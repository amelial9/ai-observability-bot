# Project Documentation Hub

This folder is a living source of truth for project context, progress, and next steps.

## Start Here

- `doc/PROJECT_OVERVIEW.md` - What this repository is, how it works today, and key components.
- `doc/PROGRESS.md` - Progress tracker across Phase 1, Phase 2, and Phase 3.
- `doc/MULTITENANCY_REVIEW.md` - Review of the multitenancy plan and implementation guidance.

## Why This Exists

This project has evolved from early architecture planning into active implementation. These docs are designed so any AI assistant (or new teammate) can quickly understand:

- current architecture and limitations,
- which roadmap phase the team is in,
- what is done vs in progress vs not started,
- and what should be implemented next.

## Update Protocol

When making meaningful changes:

1. Update `doc/PROGRESS.md` status/checklists.
2. Update `doc/PROJECT_OVERVIEW.md` if architecture or stack changed.
3. Update `doc/MULTITENANCY_REVIEW.md` when tenant-isolation design decisions change.

Keep entries short and concrete so the docs remain useful during rapid iteration.
